# Ghast Language Development Plan

## Current State Analysis
Ghast is a functional programming language implemented in Rust with:
- Parser combinator library (monapa) for syntax analysis
- Three-phase processing pipeline (phase2: FlatIR, phase3: Core language)
- Basic support for literals, variables, lambdas, tuples, and operators
- Demo functionality that parses simple expressions like "1 + 1"

## Realistic Near-Term Goals

### Phase 1: Stabilize Core Functionality (1-2 weeks)
1. **Complete the evaluation pipeline**
   - Uncomment and implement the corelang conversion in main.rs
   - Fix any remaining TODOs in phase2.rs (particularly the panic in binop())
   - Ensure proper error handling throughout the pipeline

2. **Expand literal support**
   - Add string literals (phase2.rs literal() function)
   - Add float/decimal support
   - Add boolean literals

3. **Improve operator system**
   - Document current operator precedence and fixity rules
   - Add common operators: *, /, %, ==, !=, <, >, <=, >=
   - Implement proper precedence parsing (currently seems to have issues)

### Phase 2: Enhance Usability (2-4 weeks)
1. **Standard library basics**
   - Implement essential built-in functions beyond current add/sub/neg/etc.
   - Add: length, head/tail for tuples, basic I/O, type conversion
   - Create a standard module system

2. **Improve REPL/interactive mode**
   - Add command history
   - Add multi-line input support
   - Add helpful error messages and debugging info

3. **Testing expansion**
   - Add property-based testing for parser
   - Add integration tests for complete programs
   - Benchmark performance of parsing/evaluation

### Phase 3: Practical Applications (1-2 months)
1. **Domain-specific examples**
   - Create examples showing language utility:
     - Simple configuration parser
     - Basic data transformation scripts
     - Mathematical expression evaluator
   - Document these in examples/ directory

2. **Tooling improvements**
   - Add basic IDE support (syntax highlighting via tree-sitter or similar)
   - Create formatter/pretty-printer
   - Add basic profiler

3. **Documentation**
   - Create tutorial for newcomers
   - Document language specification
   - Create API documentation for embedding Ghast in other Rust projects

## Success Metrics
- Ability to write and execute non-trivial programs (>10 lines)
- Clear error messages for common mistakes
- Performance suitable for scripting tasks (<100ms for small programs)
- Comprehensive test coverage (>80% of core functionality)

## Avoiding Overly Ambitious Goals
- No immediate plans for self-hosting compiler
- No complex type system additions (keep dynamically typed for now)
- No foreign function interface initially
- Focus on making existing features robust before adding major new ones
